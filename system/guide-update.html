Passo a Passo da Instalação

Siga estes passos cuidadosamente para colocar o JARVIS em funcionamento:

01
Criar Pasta do Projeto
Crie uma pasta na sua área de trabalho chamada "jarvis" e coloque o PDF de configuração dentro dela.
02
Abrir o Claude Code
Usando o aplicativo Claude Desktop, vá até a seção "Code" abaixo do chat e selecione a pasta Jarvis na sua área de trabalho.
03
Executar o Blueprint
Digite no chat:
"Leia este PDF inteiro e execute tudo que está descrito nele para criar o Jarvis. O PDF completo está na pasta desktop/jarvis." Inclua sua chave API na mensagem: OPENAI_API_KEY=COLE-SUA-CHAVE-AQUI
04
Verificar Autenticação
Digite no chat: Certifique-se de que o Claude CLI está autenticado com a conta do usuário e que a chave API da OpenAI fornecida está funcionando no projeto. Realize todos os testes no terminal e converse por voz via OpenAI.
05
Testar o JARVIS
Execute um teste completo do sistema JARVIS para confirmar que todas as funcionalidades estão operando corretamente, incluindo interação por voz e respostas da IA.

Dicas e Boas Práticas

Mantenha sua chave API segura e nunca a compartilhe publicamente.
Comece com o valor mínimo de créditos na OpenAI — você sempre pode adicionar mais depois.
Certifique-se de que sua assinatura do Claude está ativa antes de iniciar o processo de configuração.
Se algum passo falhar, releia as instruções e verifique suas credenciais.

Passo 06 — Diagnóstico e Otimização de Voz

Após a instalação, cole este prompt no chat do JARVIS para auditar o sistema de voz, corrigir problemas de alucinação e otimizar a velocidade de resposta.

Prompt de Diagnóstico — Cole no Chat do JARVIS
JARVIS, execute a complete voice system audit and repair. Inspect the actual source code and fix anything broken: 1. READ server.js and verify the hallucination filter exists with these rules: - Block any transcription under 4 characters - Block single words under 8 characters - Block known phantom phrases in English: "Bye", "Goodbye", "Farewell", "Thank you for watching", "Subscribe", "Like and subscribe" - Block known phantom phrases in Portuguese: "Tchau", "Adeus", "Obrigado por assistir", "Legendas pela comunidade", "Inscreva-se", "Curta e se inscreva" - Block noise markers: [Music], [Applause], (silence), "...", "Um", "Uh", "Hmm" - Block repetitive text where 70%+ of words are the same - If the filter is missing or incomplete, ADD it before the /api/stt route 2. READ server.js and verify /api/stt route has: - No temperature parameter (remove if present) - No fixed language parameter — try English first, if hallucination detected try Portuguese - Prompt hint with realistic command examples in both languages - Rejects audio files smaller than 2000 bytes - Logs raw transcription to console before and after filtering - Saves last audio to system/last-audio-debug.webm for debugging 3. READ public/script.js and verify MediaRecorder config: - getUserMedia with: echoCancellation: false, noiseSuppression: false, autoGainControl: true - NO sampleRate constraint, NO channelCount constraint - mediaRecorder.start() with NO timeslice argument (single chunk, not fragmented) - audioBitsPerSecond: 128000 - Minimum recording duration check (reject < 800ms) - Audio level monitoring with peak volume tracking - Silence detection (reject if peakVolume < 5) 4. READ public/script.js and verify transcribeAndSend handles filtered responses: - If server returns { filtered: true }, show warning in terminal and return to idle - If server returns empty text, show "No speech detected" warning 5. Save to system/JARVIS-MEMORY.md under Technical Context: "Voice filter audit complete — hallucination filter verified with EN/PT patterns, MediaRecorder set to single-chunk 128kbps WebM, no browser audio processing, dual-language STT with fallback" 6. Report: list each check as PASS or FAIL. If any FAIL, fix the code immediately. 7. OPTIMIZE RESPONSE SPEED AND BEHAVIOR: A) INSTANT CONTEXTUAL ACKNOWLEDGMENT: When the user asks JARVIS to do anything (create, build, generate, analyze, write, design, fix, make, produce, develop), JARVIS must IMMEDIATELY respond in voice AND terminal with a short contextual confirmation BEFORE starting the work. Examples: - "Create an e-book about digital marketing" → "Right away, sir. Creating your digital marketing e-book now." - "Build me a website" → "On it, sir. Setting up the website architecture." This must NOT use a separate Claude CLI spawn — use a pre-built contextual phrase generated from the user's message keywords, sent to TTS immediately while the main process works in background. B) SMART MODEL SELECTION BY COMPLEXITY: - claude-haiku-4-5-20251001 ONLY for simple questions (greetings, status checks, quick answers) - claude-sonnet-4-6 for creative and moderate tasks (create, build, generate, write, design, fix, analyze, report, PDF, website, code) - claude-opus-4-6 for complex and critical tasks (architect, refactor, e-book, infrastructure, deploy, comprehensive, advanced, production, enterprise, detailed, deep analysis) - Voice responses must be fast (TTS first 2 sentences only) but task execution uses the best model for quality - Skip semantic memory search for voice requests (saves 1-2s) - Stream response to terminal as first chunk arrives - Run storeMemory and appendHistory AFTER response (non-blocking) C) VOICE COMPLETION NOTIFICATION: When JARVIS finishes a long task, send voice: "Done, sir. Your [item] is ready." Show [system] log with path and download cards. D) REDUCE buildJarvisPrompt for voice: shorter prompt for simple questions, full context only for creation tasks. E) NO QUESTIONS — JUST EXECUTE: Add to the JARVIS system prompt: "CRITICAL: NEVER ask clarifying questions. NEVER ask 'would you like me to...' or 'should I...'. When sir gives a command, EXECUTE IT IMMEDIATELY and deliver the complete result. Make smart decisions on your own. If details are missing, use your best judgment and deliver." JARVIS must act like a real assistant — when told to do something, DO IT, don't ask about it. Implement ALL optimizations in the actual code.
Importante: Execute este diagnóstico após cada instalação nova ou se a voz parar de funcionar corretamente.
